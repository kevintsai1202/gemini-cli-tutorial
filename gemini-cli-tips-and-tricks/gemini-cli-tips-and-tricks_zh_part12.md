## 技巧 21：使用 /COPY 快速複製到剪貼簿

**快速用例：** 立即將 Gemini CLI 的最新答案或程式碼片段複製到您的系統剪貼簿，無需任何多餘的格式或行號。這非常適合快速將 AI 生成的程式碼貼到您的編輯器中，或與隊友分享結果。

當 Gemini CLI 提供答案（尤其是多行程式碼區塊）時，您通常希望在其他地方重複使用它。`/copy` 斜線命令透過將 CLI 產生的最後一個輸出直接複製到您的剪貼簿，使這變得毫不費力。與手動選取（可能會抓取行號或提示文字）不同，`/copy` 只抓取原始回應內容。例如，如果 Gemini 剛剛生成了一個 50 行的 Python 腳本，只需輸入 `/copy` 就會將整個腳本放入您的剪貼簿，準備貼上——無需捲動和選取文字。在底層，Gemini CLI 使用適用於您平台的適當剪貼簿工具（例如 macOS 上的 `pbcopy`，Windows 上的 `clip`）。一旦您運行該命令，您通常會看到確認訊息，然後您可以將複製的文字貼到您需要的任何地方。

**工作原理：** `/copy` 命令要求您的系統具有可用的剪貼簿工具。在 macOS 和 Windows 上，所需的工具（分別為 `pbcopy` 和 `clip`）通常是預先安裝的。在 Linux 上，您可能需要安裝 `xclip` 或 `xsel` 才能使 `/copy` 正常運作。確保之後，您可以在 Gemini CLI 列印答案後的任何時間使用 `/copy`。它將捕獲整個最後一個回應（即使它很長），並省略 CLI 可能在螢幕上顯示的任何內部編號或格式。這可以讓您在傳輸內容時避免處理不需要的偽影。這是一個小功能，但在您反覆運算程式碼或編譯 AI 生成的報告時，可以大大節省時間。

**專業技巧：** 如果您發現 `/copy` 命令不起作用，請仔細檢查您的剪貼簿工具是否已安裝且可存取。例如，Ubuntu 用戶應運行 `sudo apt install xclip` 以啟用剪貼簿複製。一旦設定完成，`/copy` 讓您可以零摩擦地分享 Gemini 的輸出——複製、貼上，然後就完成了。

## 技巧 22：掌握 CTRL+C 以進入 Shell 模式和退出

**快速用例：** 透過單次按鍵——以及快速雙擊以完全退出 CLI——乾淨地中斷 Gemini CLI 或退出 shell 模式，這要歸功於多功能的 `Ctrl+C` 快捷鍵。這讓您在需要停止或退出時立即獲得控制。

Gemini CLI 像 REPL 一樣運行，了解如何中斷操作至關重要。單次按下 `Ctrl+C` 將取消目前操作或清除您已開始輸入的任何輸入，本質上充當「中止」命令。例如，如果 AI 正在生成冗長的答案，而您已經看夠了，請按下 `Ctrl+C`——生成會立即停止。如果您已開始輸入提示但想丟棄它，`Ctrl+C` 將清除輸入行，以便您可以重新開始。此外，如果您處於 shell 模式（透過輸入 `!` 運行 shell 命令啟用），單次 `Ctrl+C` 將退出 shell 模式並返回到正常的 Gemini 提示符（它會向正在運行的 shell 進程發送中斷）。如果 shell 命令掛起或您只是想返回 AI 模式，這非常方便。

連續兩次按下 `Ctrl+C` 是完全退出 Gemini CLI 的快捷鍵。將其視為「`Ctrl+C` 取消，再次 `Ctrl+C` 退出」。這種雙擊會向 CLI 發出終止會話的訊號（您會看到再見訊息或程式將關閉）。這是一種比輸入 `/quit` 或關閉終端機視窗更快的替代方法，讓您可以從鍵盤優雅地關閉 CLI。請注意，如果需要清除輸入或中斷操作，單次 `Ctrl+C` 不會退出——它需要第二次按下（當提示符空閒時）才能完全退出。這種設計可以防止在您只想停止目前輸出時意外關閉會話。

**專業技巧：** 在 shell 模式下，您還可以按下 `Esc` 鍵以離開 shell 模式並返回 Gemini 的聊天模式，而無需終止 CLI。如果您喜歡更正式的退出方式，`/quit` 命令始終可用於乾淨地結束會話。最後，Unix 用戶可以在空提示符下使用 `Ctrl+D` (EOF) 退出——如果需要，Gemini CLI 將提示確認。但對於大多數情況，掌握 `Ctrl+C` 的單次和雙擊是保持控制的最快方法。
